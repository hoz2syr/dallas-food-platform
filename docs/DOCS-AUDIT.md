ملف: توثيق - تقرير تدقيق سريع
=============================

الهدف
-----
هذا المستند يقدّم ملخّص تدقيق للتوثيق الحالي في المستودع (دليل المشروع والملفات داخل `docs/`) ويقترح تحسينات قابلة للتنفيذ.

الملخّص السريع
---------------
- التوثيق العام جيد ويحتوي على ملفات معمارية (ADRs) وملفات عقود الـ API.
- يفتقر المستودع إلى جدول مركزي لمتغيرات البيئة لكل خدمة.
- بعض التطبيقات (مثل `apps/web`, `apps/customer-app`) تحتوي README محلية قصيرة؛ تحتاج ملفات Quickstart مخصّصة للتشغيل المحلي.
- لا يوجد CHANGELOG مركزي أو ملخص للتغييرات الهامة بين الإصدارات.

ملاحظات مفصّلة
----------------
1. متغيرات البيئة
   - لا يوجد ملف مرجعي مركزي يحدّد المتغيرات المطلوبة لكل خدمة مع أمثلة.
   - اقتراح: إضافة `docs/ENVIRONMENT_MATRIX.md` أو جدول داخل `docs/` يشرح: المتغير، النوع، مثال، مطلوب/اختياري، الخدمة.

2. تشغيل محلي (Developer Quickstart)
   - بعض التطبيقات تذكر خطوات عامة في `README.md` الجذري، لكن يفضل ملفات Quickstart قصيرة داخل كل `apps/*/README.md` تشرح: تشغيل dev، متغيرات محلية، والبوابات (ports).

3. ملفات العقود والعمليات
   - تأكد أن `docs/API-CONTRACTS.md` محدثة مع أمثلة الطلب/الاستجابة ونسخ OpenAPI إن وُجدت.

4. CHANGELOG و Releases
   - إضافة `docs/CHANGELOG.md` أو استخدام GitHub Releases لملخّص تغييرات الإصدارات.

تغييرات سريعة قمت بها الآن
---------------------------
- إضافة هذا التقرير: `docs/DOCS-AUDIT.md`
- ربط تقرير المراجعة في مستوى الجذر (`README.md`) كتذكير للمساهمين

خطوات مقترحة قصيرة المدى (يمكن تنفيذها تلقائياً)
-------------------------------------------------
1. إضافة `docs/ENVIRONMENT_MATRIX.md` (مهم).  
2. لكل تطبيق في `apps/` إضافة أو تحديث `README.md` بـ Quickstart.  
3. إضافة `docs/CHANGELOG.md` أو تعليمات إصدار.  

من يحتاج لمتابعة
-----------------
- أقترح أن يبدأ فريق DevOps/Platform بـ `docs/ENVIRONMENT_MATRIX.md`.  
- فريق الواجهات (Frontends) يحدثReadmes الخاصة بـ `apps/web` و `apps/customer-app`.

إذا أردت، أقوم الآن بإضافة `docs/ENVIRONMENT_MATRIX.md` أولي يتضمن أمثلة متغيراتٍ للخدمات الأساسية (postgres, redis, rabbitmq, auth).
